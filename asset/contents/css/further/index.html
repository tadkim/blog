Pointer-events

pointer-events 값을 지정하여 HTML 엘리먼트가 CSS에서의 hover/active 상태, Javascript에서의 클릭, 탭 등의 마우스/터치 이벤트들에 어떤 식으로 반응할 지를 정할 수 있다.

총 11개의 속성값이 있으며, 그중 3가지는 모든 HTML 엘리먼트에 사용하능하고 나머지는 SVG를 콘트롤 하는 데 사용된다. Mozila MDN에 의하면 SVG 엘리먼트가 아닌 경우에 pointer-events의 쓰임은 아직 실험 중이고 CSS4에서 본격화될 것 같다고 한다.

pointer-events의 기본 값은 auto이며, 자식 엘리먼트에 상속되는 값이다.

사용하는 경우

  • 사용자가 볼 수 있는 엘리먼트에 대해 포인터 이벤트를 없애고 싶을 때.
  • 사용자가 볼 수 없는 엘리먼트에 대해 포인터 이벤트를 지정하고 싶을 때.
  • 투명하거나, 마스크 영역 밖에 있거나 등의 이유로 보이지 않는 영역에 대해 포인터 이벤트를 지정하고 싶을 때.
  • 포인터 이벤트가 있는 다른 엘리먼트와 중첩될때 (i.e., position이 fixed된 엘리먼트를 가졌을 경우) 우선 순위를 정하기 위해

모든 HTML 엘리먼트에 적용 가능한 3가지 속성값

  1. none : 클릭은 물론이고, hover/active 상태나 커서 옵션 등을 모두 없앤다. 툴팁을 만들 때 사용되기도 한다. 이 none 값을 쓴다는 것은 none값이 지정된 엘리먼트 대신에 그 아래에 있는 엘리먼트에 대해 클릭, hover 등을 타겟팅 시키라는 의미이다. 예를 들어, 툴팁의 배경을 반투명하게 해도 툴팁을 띄움으로 인해서 툴팁이 있는 자리 아래의 텍스트가 긁히지 않는 경우에, 툴팁의 pointer-events: none;으로 해줌으로서 z-index상으로 아래에 있는 텍스트를 긁히게 해줄 수 있다. 상속되는 값이지만 자식 엘리먼트를 none이 아닌 값으로 명확히 지정한다면 trigger기능도 충분히 가능하다.
  2. auto : 부모가 pointer-events: none;으로 된 경우 자식 엘리먼트에 많이 쓴다. 무력화된 클릭 등의 기능을 다시 기본으로 살려준다. 만약 엘리먼트에 click event listener를 추가했다면 pointer-events값을 지우거나 auto로 바꾸어 주어야한다. 기본적으로, click event는 pointer-events값을 준수하기 때문이다. SVG에 지정할 경우에 auto 값은 visiblePainted 값과 동일한 효과를 낸다.
  3. inherit : 부모의 pointer-events 속성을 상속한다.

SVG only 속성값

이렇게 나누는 것에 대해 명확하지 않다며 욕을 들어먹고 있긴 하지만.

  1. visiblePainted : visible로 지정되어 있는 내부값(i.e.,fill)이나 주변값(i.e.,storke)이 none이 아니라 실제로 값이 지정되어 있는 경우에 pointer-events를 지정하는 것.
  2. visibleFill : visible로 지정되어 있는 내부값(i.e.,fill)에 대해 pointer-events를 지정하는 것.
  3. visibleStroke : visible으로 지정되어 있는 주변값(i.e.,stroke)에 대해 pointer-events를 지정하는 것.
  4. visible : visible하게 지정되어 있는 내부값(i.e.,fill)이나 주변값(i.e.,stroke)에 대해 pointer-events를 지정하는 것.
  5. painted : 엘리먼트가 visible로 설정되어있든말든, 내부값이나 주변값이 none이 아닌 다른 것으로 지정된 경우에 pointer-events를 지정하는 것.
  6. fill : 엘리먼트가 visible로 설정되어있든말든, 내부값이 none이 아닌 다른 것으로 지정되어 있다면 pointer-events를 지정함.
  7. stroke : 엘리먼트가 visible로 설정되어있든말든, 주변값이 none이 아닌 다른 것으로 지정되어 있다면 pointer-events를 지정함.
  8. all : fill,stroke값, 그리고 visible 여부와 상관없이 pointer-events를 지정하는 것.

추천되는 속성값들

none, painted, all. SVG의 경우 fill과 stroke에 대한 값들을 세분화 하는 게 적절할 수 있다. 위의 세 속성값은 painted, paintedFill,paintedStroke와는 달리 투명도, 가시성(visibility)은 무시하고 지정되는 속성값이다. 그리고 fill이나 stroke가 아닌 background나 border값을 쓰는 HTML 박스 모델에 더 적합하다.
