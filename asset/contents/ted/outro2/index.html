마무리하며 : 간단 회고

TED 프로젝트를 진행과정 차트 및 간단 회고 (리스트 형식)

간단 회고

좋았던 점

  • SVG, CSS animation 등 궁금하거나 필요한 게 생겼을 때 미루지 않고 공부하고 해결할 수 있던 점
  • 변수를 넣었을 때 형태가 나오는 공식을 제작해본 경험
  • 더 이상 얻어갈 것이 없는 적절한 시점에 프로젝트를 끝낸 것
  • 아카이빙을 잘 해둔 것
  • 방향성이나 효율성에 대해 탬과 계속 솔직하게 얘기한 것
  • 데이터 크롤링이나 정제에 대한 부분을 탬에게 믿고 맡길 수 있는 점
  • 깊이 탐구해보고 정리해가면서 학습할 수 있어서 좋았음
  • 내부 프로젝트가 외주를 받아 할 때 생각보다 더 도움이 많이 될 것 같음

아쉬웠던 점, 개선점

  • 전체적인 방향성이나 합의된 사항들에 대해 헷깔릴 때마다 보고 리마인드 할 수 있게 크게 적어서 벽에 붙여놓거나 했으면 좋겠음
  • 디자인 가이드 라인을 알고리즘화 하여 명확하게 제시하지 못한 점
  • 생활 패턴이 불규칙적이었던 점
  • 하루 이틀 단위가 아닌 3-4일 단위로 마일스톤을 정하면 좋겠음
  • 나는 데이터 분석 경험이 워낙 전무하다보니 데이터로 EDA를 진행할 수 없었음, 통계를 조금 공부하다보니 제대로 하려면 굉장히 공부할 것이 많아서 부담스러웠음

인사이트

  • 데이터의 분포에 대해 먼저 파악하고 형태 제작에 들어가야 효율적이겠다.
  • 데이터를 바인딩하여 형태를 콘트롤 하는 부분과 아닌 부분을 먼저 정하자.

생겨난 질문들

  • 통계, 데이터 분석을 앞으로 어떤 식으로 할 수 있을 지

과제로 남은 것들

  • HTML/CSS로 Animated Chart 여러 개 만들고 탬플릿화 하기
  • 데이터 시각화 전문가로서의 데이터 해석하는 경험, 친밀도
  • Arc의 sweep-flag 속성 다시 공부
  • 과정들 중 공유할 가치 있는 것들을 우리가 추구하는 느낌으로, 소화하여 블로그에 공유하는 것
  • UX, 마이크로 인터랙션 부분에서 연구

윰의 회고를 바탕으로 탬과 이야기

아카이빙 관련

  • 탬 : SVG나 CSS animation을 다루면서 봉착했던 문제점과 그 해결방법 등을 잘 모아두면 좋을 듯 하다. 코드펜에라든지 제목을 잘 분류하여 재사용이나 검색이 가능하게 하면 나중에 까먹어도 찾기 쉬울 것이다.
  • 탬 : 아카이빙 한 것도 재사용성에 대해 이야기를 나눠봐야할 것 같다.
  • 윰 : 이번 작업 과정에서 아카이빙을 하는 것이 의미는 있지만 상당히 고된 일이었다. 앞으로 프로젝트들은 아마 아카이빙에 긴 시간을 중간에 투입하지는 못할 듯 하다. 아카이빙 한 것을 후에 찾아보게 될 때 어떤 것을 주로 찾아보게 될까 생각해보고 중요한 것을 필수적으로 아카이빙하고 상대적으로 중요도가 떨어지는 것들은 간략하게 정리해두도록 해야할 것 같다.
  • 캘린더에 포스트잇 붙이는 것을 그대로 해되 잔디에 좀더 상세한 작업 + 걸린 시간을 적어두면 좋을 것 같다. 시간 단위로 파악되면 후에 어떤 일에 많은 시간이 투입되었는 지 계획잡을 때 참고하기 좋을 것이다.

크롤링, 정제 관련

  • 탬 : 크롤링과 정제에 기술적인 어려움은 없었지만 시간이 많이 걸린다. 이번에도 일주일 걸렸는데 이는 전체 프로젝트 기간의 1/4에 해당하는 기간이다. 수집/정제/기획에 부하가 크다고 느껴진다.
  • 윰: 기획과 정제에 너무 많은 시간이 투입되어 문제라고 보기보다는 기획과 정제가 원래 그만큼 걸려야하는 작업이라고 보면 될 것 같다. 그리고 외주를 받거나 해서 타임라인을 계획할 때 기획과 정제 시간을 우리가 기존에 생각했던 것 보다 길게 잡아야 한다는 의미로 보인다.
  • 윰, 탬: 크롤링은 함께 하지 않는 게 좋을 것 같지만 노가다성 작업은 분업하기로 하자.

통계 공부 관련

  • 탬 : 통계 공부, 데이터 분석 관련해서는 월 스트리트 저널 책을 보면서 기본 개념을 틀리지 않도록 노력하면 될 것 같다. 증감률/비율/대비 등의 개념을 스터디 하면 좋겠다.
  • 윰 : 의도치 않게 몰라서 하는 왜곡을 하지 않기를 바란다. 통계의 미학 책을 같이 완독했으면 한다.

데이터 리터러시 관련

  • 탬 : 윰은 스스로 어떤 부분에서 데이터 리터러시가 약하다고 느끼는지 모르겠다. 더치트나 이주노동자 프로젝트 등 할 때 오히려 잘하는 점이라 느꼈다. 자기 도메인 영역이 아닌 곳에서 늘 사전 지식이나 데이터의 한계를 느끼는 것은 당연한 일인지도. 프로젝트를 할 때 경험도 중요하고 해당 도메인의 현장 사람들과 대화를 잘 하는 것이 중요할 것 같다.
  • 윰 : 막연히 자신감이 없는 것 같다. 그런데 전에도 말했다시피 데이터 자체에 대한 흥미는 없는 편인 듯 하다.
  • 윰,탬 : 사실 자체 프로젝트가 아닌 클라이언트 잡에서 Insight는 답정너에 가깝겠다. 데이터를 보고 분석하는 쪽은 컨설팅에 가깝다고 생각한다.

생활 패턴 관련

  • 윰 : 생활 패턴이 점점 과도한 작업이 되기도 했고, 인풋을 위한 시간이 너무 줄어들었다. 눈 뜨면 출근 퇴근 후 거의 바로 자기 때문이다. 가고싶던 전시가 끝나서 아쉽다.
  • 탬 : 인풋은 중요하다는 것에 동의한다.
  • 윰 : 포항갔을 때 전화상으로도 이야기했지만 시간 분할과 개인적인 시간 운용을 위해 시간을 따로 잡는 것은 꼭 필요하다.
  • 윰, 탬 : 맞다. 우리의 성향상 자유에 맡기면 작업을 더 하려고 할 것이다. 일주일에 하루는 꼭 요일 정해서 쉬는 것으로. 지키지 않으면 벌칙을 !!

마일스톤 관련

  • 윰, 탬 : 작은 방향, 목표가 합의될 때마다 일단 짧은 기간을 목표로 세팅하자. 엄격한 것은 아니고 해가면서 줄일 수도 늘일 수도 있는 시간계획이지만 이를 통해 각자 시간 운용에 유도리가 생길 것이다. 달릴 때 달릴 명분도 생기고.
